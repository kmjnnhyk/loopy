# Security Policy

## Supported versions

loopy is pre-1.0. Only the latest `0.1.x` release (published under the `next`
dist-tag) receives security fixes.

| Version | Supported |
|---|---|
| `0.1.x` (latest) | ✅ |
| older `0.1.x`   | ❌ (upgrade to latest) |
| `< 0.1.0`        | ❌ |

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.**

Report privately using GitHub's private vulnerability reporting:

➡️ **[Report a vulnerability](https://github.com/kmjnnhyk/loopy/security/advisories/new)**
(repo **Security** tab → **Advisories** → **Report a vulnerability**)

This opens a private channel visible only to you and the maintainer, where a fix
and a coordinated disclosure can be prepared before anything is public.

Please include:

- the affected package(s) (`@loopyjs/core`, `@loopyjs/anthropic`, …) and version,
- a description of the issue and its impact,
- a minimal proof-of-concept or reproduction, if you have one.

## What to expect

- **Acknowledgement** within a few days of your report.
- An assessment of severity and, if confirmed, a fix on the supported release.
- **Coordinated disclosure:** we'll agree on a timeline before public disclosure,
  and credit you in the advisory unless you prefer to remain anonymous.

Thank you for helping keep loopy and its users safe.
