# loopy docs

The documentation site for [loopy](https://github.com/kmjnnhyk/loopy) — "React for agents," a type-safe TypeScript DSL for LLM agents, tools, workflows, and teams. Built with [Astro](https://astro.build) + [Starlight](https://starlight.astro.build).

This directory is self-contained: its own `package.json`, its own `node_modules` (not symlinked to the parent repo), and its own build. It does not affect the root project's `tsc` gates (`tsconfig.json`, `tsconfig.examples.json`, `tsconfig.negative.json`).

## Commands

Run from `website/`:

| Command | Action |
| --- | --- |
| `npm install` | Install dependencies |
| `npm run dev` | Start the local dev server at `localhost:4321` |
| `npm run build` | Build the production site to `./dist/` |
| `npm run preview` | Preview the built site locally |

## Content

Pages live in `src/content/docs/`, one Markdown/MDX file per route. The sidebar structure is configured explicitly in `astro.config.mjs` (not autogenerated), so a new page needs an entry there to show up in navigation.

**Accuracy rule:** every code sample here should trace back to `src/index.ts` or `examples/*.ts` in the parent repo. `team()`-related content is sourced from the `feat/team-type-surface` branch, which hasn't merged into `master` yet — those pages say so explicitly. See [Status & Roadmap](./src/content/docs/status-roadmap.md) in the site itself for the up-to-date picture.

## Deployment

A GitHub Actions workflow (`.github/workflows/deploy-docs.yml` at the repo root) builds this site and publishes it to GitHub Pages on push to `master`. **GitHub Pages itself is not yet enabled for the repository** — enabling it (Settings → Pages) is a deliberate, separate, outward-facing step left for a maintainer to take after review.

If this site is ever served from a subpath (e.g. a GitHub Pages *project* page at `https://kmjnnhyk.github.io/loopy/`), see the note at the top of `astro.config.mjs` first — the docs' internal links are currently root-relative and assume the site is served at `/`.
